# kalmbuddha — Интересные места Элисты (MVP с регистрацией)

Минимальный рабочий проект: **React (Vite) + Node.js/Express + PostgreSQL (Sequelize) + JWT**.

## Быстрый старт (локально)

### 1) Требования
- Node.js 18+ (или 20+)
- PostgreSQL 14+ (локально установленный)

### 2) Настройка базы
Создай БД и пользователя в Postgres (пример):
- База: `kalmbuddha`
- Пользователь: `kalmbuddha_user`
- Пароль: `kalmbuddha_pass`

### 3) Настройка переменных окружения (server)
Скопируй файл `server/env.example` в `server/.env` и поправь значения под себя.

### 4) Установка зависимостей
В корне проекта:

```bash
npm --prefix server i
npm --prefix client i
```

### 5) Запуск
В двух окнах терминала:

```bash
npm --prefix server run dev
```

```bash
	npm --prefix client run dev
```

По умолчанию:
- API: `http://localhost:4000`
- Frontend: `http://localhost:5173`

> При необходимости на фронтенде можно задать `VITE_API_URL`, например `http://localhost:4000/api`.

### Загрузка файлов
- Эндпоинт: `POST /api/uploads` (требует авторизацию, поле `files` FormData, можно до 5 файлов).
- Файлы кладутся в `server/uploads/`, доступны по URL `/uploads/<имя>`.
- На фронте в форме места можно загружать фото с компьютера/телефона — ссылки подставляются автоматически.

## Развёртывание на VPS

### Минимальные требования к серверу
| Ресурс | Минимум | Рекомендуется |
|--------|---------|---------------|
| **RAM** | 1 GB | 2 GB |
| **CPU** | 1 ядро | 2 ядра |
| **Диск** | 10 GB SSD | 20 GB SSD |
| **ОС** | Ubuntu 22.04 LTS | Ubuntu 22.04 LTS |

**На что смотреть при покупке:**
- **RAM** — Node.js + PostgreSQL + Nginx потребляют ~500–800 MB в простое. 1 GB хватит для небольшой нагрузки; при 100+ одновременных пользователях лучше 2 GB.
- **CPU** — 1 ядро достаточно для MVP; 2 ядра дадут запас при пиках.
- **Диск** — SSD обязателен (быстрее БД и загрузки). Учитывай: фото хранятся в `uploads/`, со временем объём растёт.
- **Трафик** — обычно безлимитный на тарифах 1–2 GB RAM.

**Провайдеры (РФ/СНГ):** Selectel, Timeweb Cloud, REG.RU, Beget. За рубежом: Hetzner, DigitalOcean, Vultr.

### Стек на VPS
- **Nginx** — reverse proxy, статика, SSL
- **PM2** — запуск Node.js сервера
- **PostgreSQL** — база данных
- **Node.js** — API (из `server/`)
- **Статика** — собранный фронт (`client/dist`) раздаётся через Nginx

### Переменные окружения в продакшене
- `NODE_ENV=production`
- `DATABASE_URL` — строка подключения к PostgreSQL
- `JWT_SECRET` — длинная случайная строка (минимум 32 символа)
- `CORS_ORIGIN` — домен фронтенда (например `https://elista.example.com`)
