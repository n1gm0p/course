# 🔐 Руководство по модерации и админ-панели

## Назначение администратора

Чтобы сделать пользователя с почтой `nlololo294@gmail.com` администратором:

1. Убедитесь, что пользователь зарегистрирован на сайте
2. Запустите команду в папке `server`:

```bash
npm run make-admin
```

Или вручную:

```bash
cd server
node scripts/make-admin.js
```

После этого пользователь `nlololo294@gmail.com` получит права администратора.

## Работа в двух вкладках

**⚠️ Важно:** `localStorage` общий для всех вкладок одного домена. Поэтому если вы войдете в одной вкладке, это повлияет на все остальные вкладки того же домена.

### Способы тестирования с двумя разными пользователями:

#### Способ 1: Обычная вкладка + Режим инкогнито (Рекомендуется)

1. **Вкладка 1 (Обычный пользователь):**
   - Откройте сайт в обычной вкладке
   - Зарегистрируйтесь или войдите как обычный пользователь
   - Создайте место или комментарий
   - Они будут отправлены на модерацию

2. **Вкладка 2 (Администратор):**
   - Откройте сайт в режиме инкогнито (Ctrl+Shift+N в Chrome/Edge, Ctrl+Shift+P в Firefox)
   - Войдите как `nlololo294@gmail.com`
   - В навигации появится ссылка "Модерация"
   - Перейдите в админ-панель и одобрите/отклоните контент
   - Режим инкогнито использует отдельный localStorage

#### Способ 2: Два разных браузера

- Откройте сайт в Chrome как обычный пользователь
- Откройте сайт в Firefox (или Edge) как администратор
- Каждый браузер имеет свой собственный localStorage

#### Способ 3: Два профиля браузера

- Chrome: используйте разные профили пользователя
- Каждый профиль имеет свой localStorage

### Как проверить функционал:

1. **Вкладка 1 (Обычный пользователь):**
   - Откройте сайт
   - Зарегистрируйтесь или войдите как обычный пользователь
   - Создайте место или комментарий
   - Они будут отправлены на модерацию

2. **Вкладка 2 (Администратор - в режиме инкогнито или другом браузере):**
   - Откройте сайт в режиме инкогнито/другом браузере
   - Войдите как `nlololo294@gmail.com`
   - В навигации появится ссылка "Модерация"
   - Перейдите в админ-панель и одобрите/отклоните контент

## Админ-панель

### Доступ:
- URL: `/admin`
- Доступна только для пользователей с правами администратора
- Ссылка "Модерация" появляется в навигации только для админов

### Функционал:

#### 1. Модерация мест
- Просмотр всех мест со статусами: `pending`, `approved`, `rejected`
- Фильтрация по статусу
- Одобрение места (кнопка "✅ Одобрить")
- Отклонение места (кнопка "❌ Отклонить") с указанием причины

#### 2. Модерация комментариев
- Просмотр всех комментариев со статусами
- Фильтрация по статусу
- Одобрение/отклонение комментариев
- Просмотр фото в комментариях

## Система модерации

### Статусы контента:

1. **`pending`** - На модерации (по умолчанию для нового контента)
2. **`approved`** - Одобрено (появляется в каталоге)
3. **`rejected`** - Отклонено (не показывается пользователям)

### Что требует модерации:

1. **Новые места:**
   - При создании получают статус `pending`
   - Проверяется на дубликаты по названию
   - После одобрения появляются в каталоге

2. **Комментарии:**
   - При создании получают статус `pending`
   - Могут содержать фото
   - После одобрения видны всем пользователям

### Что видно пользователям:

- **Обычные пользователи:** видят только одобренный контент (`status: 'approved'`)
- **Авторы:** видят свои места/комментарии независимо от статуса
- **Администраторы:** видят весь контент и могут его модерировать

## Комментарии

### Добавление комментария:

1. Откройте место (клик по карточке)
2. Прокрутите вниз до секции "Комментарии"
3. Напишите текст комментария
4. (Опционально) Добавьте фото
5. Нажмите "Отправить"
6. Комментарий отправится на модерацию

### Фото в комментариях:

- Можно добавить несколько фото
- Фото также требуют модерации
- После одобрения фото видны всем

## Проверка дубликатов

При создании места система проверяет:
- Существует ли уже место с таким же названием (регистронезависимо)
- Учитываются только места со статусом `pending` или `approved`

Если дубликат найден, пользователь получит сообщение:
> "Место с таким названием уже существует. Проверьте каталог или дождитесь модерации вашего предыдущего предложения."

## Безопасность

- Только администраторы могут модерировать контент
- Обычные пользователи не видят отклоненный контент
- Авторы видят статус своих мест/комментариев
- Все действия логируются через API

## Устранение проблем

### Пользователь не может стать админом:
1. Убедитесь, что пользователь зарегистрирован
2. Проверьте правильность email в скрипте
3. Запустите скрипт из папки `server`

### Админ-панель не открывается:
1. Проверьте, что пользователь действительно админ (в БД `isAdmin = true`)
2. Выйдите и войдите снова, чтобы обновить токен
3. Проверьте консоль браузера на ошибки

### Комментарии не появляются:
1. Проверьте, что комментарий одобрен администратором
2. Обновите страницу
3. Проверьте статус комментария в админ-панели
